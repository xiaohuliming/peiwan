# 嗯呢呗电竞 - 陪玩店管理系统 开发文档

## 一、项目概述

### 1.1 项目简介
嗯呢呗电竞陪玩店综合管理系统，包含**Web后台管理面板**和**KOOK机器人**两大模块。系统服务于电竞陪玩业务全流程，覆盖用户管理、订单管理、财务管理、数据统计、KOOK频道播报等核心场景。

### 1.2 技术栈

| 层级 | 技术选型 | 说明 |
|------|---------|------|
| 后端框架 | Flask 3.x | 轻量级Python Web框架 |
| 数据库 | MySQL 8.0 | 关系型数据库 |
| ORM | SQLAlchemy + Flask-SQLAlchemy | 对象关系映射 |
| 数据库迁移 | Flask-Migrate (Alembic) | 数据库版本管理 |
| 用户认证 | Flask-Login + Flask-JWT-Extended | Session认证(Web) + JWT(API/机器人) |
| 表单验证 | WTForms / Marshmallow | 请求数据校验与序列化 |
| 前端模板 | Jinja2 + Bootstrap 5 + Vue.js 3 (CDN) | 模板渲染 + 动态交互 |
| 文件存储 | 本地存储 / 阿里云OSS (可选) | 头像、收款码等图片 |
| 缓存 | Redis | Session存储、缓存热点数据 |
| 任务队列 | Celery + Redis | 异步任务(播报推送、定时结算) |
| KOOK SDK | khl.py / 自封装HTTP API | KOOK机器人交互 |
| Excel导出 | openpyxl | 数据导出功能 |
| 部署 | Gunicorn + Nginx | 生产环境部署 |

## 二、数据库设计

### 2.1 ER关系总览

```
User(用户) ──1:N── Order(订单)         [作为老板]
User(用户) ──1:N── Order(订单)         [作为陪玩]
User(用户) ──1:N── Order(订单)         [作为派单客服]
User(用户) ──1:N── GiftOrder(礼物订单) [作为老板]
User(用户) ──1:N── GiftOrder(礼物订单) [作为陪玩]
User(用户) ──1:N── BalanceLog(余额日志)
User(用户) ──1:N── CommissionLog(佣金日志)
User(用户) ──1:N── WithdrawRequest(提现申请)
User(用户) ──1:N── Attendance(打卡记录)
User(用户) ──M:N── User(亲密度)        [通过Intimacy关联表]
Project(项目) ──1:N── ProjectItem(子项目)
ProjectItem(子项目) ──1:N── Order(订单)
Gift(礼物配置) ──1:N── GiftOrder(礼物订单)
```

```sql
CREATE TABLE users (
    id              BIGINT UNSIGNED AUTO_INCREMENT PRIMARY KEY,

    password_hash   VARCHAR(255) NOT NULL COMMENT '密码哈希',
    
    -- 身份信息

                    COMMENT '角色: god=老板, player=陪玩, staff=客服, admin=管理员, superadmin=高级管理员',
    nickname        VARCHAR(100) DEFAULT NULL COMMENT '客户昵称(老板显示名)',
    player_nickname VARCHAR(100) DEFAULT NULL COMMENT '陪玩昵称',
    avatar          VARCHAR(500) DEFAULT NULL COMMENT '头像URL',
    badge           VARCHAR(50) DEFAULT NULL COMMENT '铭牌显示文字',
    user_code       VARCHAR(20) DEFAULT NULL UNIQUE COMMENT '用户编号(用户00001格式)',
    
    -- KOOK关联
    kook_id         VARCHAR(50) DEFAULT NULL COMMENT 'KOOK用户ID',
    kook_username   VARCHAR(100) DEFAULT NULL COMMENT 'KOOK用户名(如 小酥#0886)',
    kook_bound      TINYINT(1) NOT NULL DEFAULT 0 COMMENT 'KOOK绑定状态',
    
    -- 微信关联
    wechat_openid   VARCHAR(100) DEFAULT NULL COMMENT '微信OpenID',
    wechat_bound    TINYINT(1) NOT NULL DEFAULT 0 COMMENT '微信绑定状态',
    
    -- 财务
    m_coin          DECIMAL(12,2) NOT NULL DEFAULT 0.00 COMMENT 'M币余额(充值货币,用于消费)',
    m_coin_gift     DECIMAL(12,2) NOT NULL DEFAULT 0.00 COMMENT 'M币赠金余额(消费不加经验)',
    m_bean          DECIMAL(12,2) NOT NULL DEFAULT 0.00 COMMENT 'M豆余额(陪玩收益,可提现)',
    m_bean_frozen   DECIMAL(12,2) NOT NULL DEFAULT 0.00 COMMENT 'M豆冻结金额(可见不可提现)',
    
    -- 等级经验
    experience      INT UNSIGNED NOT NULL DEFAULT 0 COMMENT '经验值(消费充值余额1:1)',
    vip_level       VARCHAR(50) NOT NULL DEFAULT 'GOD' COMMENT 'VIP等级名称',
    vip_discount    DECIMAL(5,2) NOT NULL DEFAULT 100.00 COMMENT 'VIP折扣(%)',
    
    -- 播报设置
    anonymous_recharge  TINYINT(1) NOT NULL DEFAULT 0 COMMENT '充值播报匿名',
    anonymous_gift_send TINYINT(1) NOT NULL DEFAULT 0 COMMENT '送礼物冠名匿名',
    anonymous_gift_recv TINYINT(1) NOT NULL DEFAULT 0 COMMENT '收礼物冠名匿名',
    anonymous_upgrade   TINYINT(1) NOT NULL DEFAULT 0 COMMENT '升级播报匿名',
    broadcast_channel   VARCHAR(100) DEFAULT NULL COMMENT '专属播报频道ID',
    
    -- 其他
    referrer_id     BIGINT UNSIGNED DEFAULT NULL COMMENT '推荐人ID',
    tags            VARCHAR(500) DEFAULT NULL COMMENT '用户标签(JSON数组)',
    benefits        VARCHAR(500) DEFAULT NULL COMMENT '权益信息(JSON)',
    status          TINYINT(1) NOT NULL DEFAULT 1 COMMENT '状态: 1=正常, 0=禁用',
    register_type   ENUM('kook','wechat','manual') NOT NULL DEFAULT 'kook' COMMENT '注册方式',
    
    created_at      DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
    updated_at      DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    
    INDEX idx_kook_id (kook_id),
    INDEX idx_role (role),
    INDEX idx_nickname (nickname),
    INDEX idx_player_nickname (player_nickname),
    FOREIGN KEY (referrer_id) REFERENCES users(id) ON DELETE SET NULL
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci COMMENT='用户表';
```

### 3.3 权限装饰器设计

```python
# app/utils/decorators.py

from functools import wraps
from flask import abort
from flask_login import current_user

def role_required(*roles):
    """角色权限装饰器，支持多角色"""
    def decorator(f):
        @wraps(f)
        def decorated_function(*args, **kwargs):
            if not current_user.is_authenticated:
                abort(401)
            if current_user.role not in [r.value for r in roles]:
                abort(403)
            return f(*args, **kwargs)
        return decorated_function
    return decorator

def staff_required(f):
    """客服及以上权限"""
    return role_required(Role.STAFF, Role.ADMIN, Role.SUPERADMIN)(f)

def admin_required(f):
    """管理员及以上权限"""
    return role_required(Role.ADMIN, Role.SUPERADMIN)(f)

def superadmin_required(f):
    """仅高级管理员"""
    return role_required(Role.SUPERADMIN)(f)
```

## 六、核心业务规则详解

### 6.1 货币体系

| 货币类型 | 标识 | 用途 | 来源 | 特殊规则 |
|---------|------|------|------|---------|
| M币(充值余额) | `m_coin` | 消费、下单、送礼物 | 充值 | 消费可增加经验值(1:1) |
| M币(赠金余额) | `m_coin_gift` | 消费、下单、送礼物 | 赠金操作 | 消费**不可**增加经验值 |
| M豆 | `m_bean` | 陪玩收益、提现 | 接单完成、礼物收入 | 冻结部分不可提现 |
| M豆(冻结) | `m_bean_frozen` | 展示 | 护航/代练自动冻结 | 可见不可提现 |
| 经验值 | `experience` | VIP等级提升 | 充值余额消费 | 比例1:1 |

**消费扣除优先级**: 先扣充值余额(`m_coin`), 再扣赠金余额(`m_coin_gift`)。这保证充值余额消费能正确累加经验值。

### 6.2 订单号生成规则

格式: `YYYYMMDDHHmmssXXXX`
- 前14位: 时间戳(年月日时分秒)
- 后4位: 随机数字(避免同秒冲突)
- 示例: `20260220170124 7147`

```python
import datetime, random

def generate_order_no():
    now = datetime.datetime.now()
    rand = random.randint(1000, 9999)
    return now.strftime('%Y%m%d%H%M%S') + str(rand)
```

### 6.3 订单价格计算

**正常陪玩订单**:
```
单价 = 基础单价 + 补充单价 + 附加项目单价
订单总价 = 单价 × 游戏时长 × (老板折扣 / 100)
陪玩收益 = 订单总价 × (分红比例 / 100)
店铺收益 = 订单总价 - 陪玩收益
```

**护航/代练订单**:
```
单价 = 基础单价(默认1) + 补充单价
订单总价 = 单价 × 时长/数量
陪玩收益 = 订单总价 × (分红比例 / 100)  [自动冻结]
店铺收益 = 订单总价 - 陪玩收益
```

**礼物订单**:
```
订单总价 = 礼物价值 × 数量
陪玩收益 = 订单总价 × (分红比例 / 100)
客服分红 = 订单总价 × 1%
店铺收益 = 订单总价 - 陪玩收益 - 客服分红
```

### 6.4 游戏时长计算规则

陪玩在申报页面手动填写时间段:
- 格式: `xx月xx日xx点xx分 - xx月xx日xx点xx分`
- 系统自动计算时长(小时)
- 超过15分钟的部分直接算0.5小时
- 示例: 2小时20分 → 2.5小时; 2小时10分 → 2小时

```python
def calc_duration_hours(start_time, end_time):
    delta = end_time - start_time
    total_minutes = delta.total_seconds() / 60
    hours = int(total_minutes // 60)
    remaining_minutes = total_minutes % 60
    if remaining_minutes > 15:
        return hours + 0.5
    return float(hours)
```

### 6.5 亲密度规则

- 每完成一笔订单, 老板与陪玩之间亲密度 += 订单总价
- 每赠送一次礼物, 老板与陪玩之间亲密度 += 礼物总价
- 退款时自动扣除对应亲密度
- 管理员可手动修改/删除亲密度
- 支持按年度清空(一般一年期限)

### 6.6 经验值与VIP等级

- 仅**充值余额**(`m_coin`)消费时增加经验值, 比例1:1
- 赠金余额消费不增加经验值
- 经验值达到阈值自动升级, 需在`vip_levels`配置表中预设各等级门槛
- 升级后自动享受对应折扣

## 十、安全设计

### 10.1 认证安全

- 密码使用 `werkzeug.security.generate_password_hash` (pbkdf2:sha256) 加密存储
- Session使用Redis存储, 设置合理过期时间
- KOOK申报/确认链接使用一次性Token(JWT, 有效期24h)
- CSRF保护: Flask-WTF自带CSRF Token

### 10.2 权限安全

- 所有API接口使用权限装饰器校验
- 前端菜单根据角色动态渲染, 隐藏无权限功能
- 敏感操作(删除/退款/发放/权限变更)需二次确认弹窗
- 所有敏感操作记入 `operation_logs`

### 10.3 数据安全

- SQL注入防护: 使用SQLAlchemy ORM, 参数化查询
- XSS防护: Jinja2默认自动转义
- 文件上传: 限制文件类型(图片), 限制大小, 重命名存储
- Bot Token等敏感信息通过环境变量配置, 不入版本控制

### 10.4 操作日志

以下操作自动记录日志:
- 冻结/解冻订单
- 创建/删除订单
- 退款操作
- 充值/扣款/赠金操作
- 修改经验值
- 权限变更
- 密码重置
- 删除用户
- 清空亲密度数据

## 十四、注意事项与技术要点

### 14.1 并发安全

- 余额操作使用数据库行锁(`SELECT ... FOR UPDATE`)或乐观锁(版本号)防止并发扣款
- 订单状态变更使用状态机模式, 防止非法状态转换
- 提现操作需原子性: 冻结M豆 + 创建申请 在同一事务内完成

### 14.2 性能优化

- 排行榜等统计查询使用Redis缓存, 设置合理过期时间
- 列表页使用分页查询(每页10/20/50条可选)
- 导出大数据量时使用流式导出, 避免内存溢出
- 数据库索引: 重点优化 `created_at`、`status`、外键字段的索引

### 14.3 用户编号规则

陪玩注册后生成编号: `用户00001` 格式
```python
def generate_user_code():
    last_user = User.query.order_by(User.id.desc()).first()
    next_num = (last_user.id if last_user else 0) + 1
    return f'用户{next_num:05d}'
```

### 14.4 时区处理

所有时间统一使用 `Asia/Shanghai` 时区, 数据库存储UTC时间, 展示时转换为北京时间。

### 14.5 消费扣除规则(重要)

```python
def deduct_balance(user, amount):
    """扣除M币, 优先扣充值余额, 返回经验值增量"""
    exp_gain = 0
    remaining = amount
    
    # 先扣充值余额
    if user.m_coin > 0:
        coin_deduct = min(user.m_coin, remaining)
        user.m_coin -= coin_deduct
        remaining -= coin_deduct
        exp_gain += coin_deduct  # 充值余额消费增加经验
    
    # 再扣赠金余额
    if remaining > 0:
        if user.m_coin_gift >= remaining:
            user.m_coin_gift -= remaining
            remaining = 0
        else:
            raise InsufficientBalanceError("余额不足")
    
    # 增加经验值
    user.experience += int(exp_gain)
    check_vip_upgrade(user)
    
    return exp_gain
```
