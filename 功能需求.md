# **电竞陪玩店 - 机器人&后台管理面板开发需求文档**

## **一、系统概述**

本系统为陪玩店的综合管理后台与机器人系统，支持 **老板（客户）** 、 **陪玩** 、 **客服** 、 **管理员** 、**高级管理员**五种角色，实现订单管理、财务管理、用户管理、数据统计等全流程业务闭环。

---

## **二、角色体系与权限**

### **表格**复制

| **角色**   | **权限级别** | **核心权限**                                               |
| ---------------- | ------------------ | ---------------------------------------------------------------- |
| GOD（老板/客户） | 普通用户           | 下单、查看订单、充值、提现、收礼物                               |
| 小猪崽（陪玩）   | 普通用户           | 接单、申报订单、提现、收礼物、查看收益、查看消费-接单-亲密度排行 |
| 客服             | 中级权限           | 派单、用户管理、订单操作、充值赠金、打卡                         |
| 管理员           | 高级权限           | 全部客服权限 + 提现审批、数据导出、系统配置、礼物管理、播报管理  |
| 高级管理员       | 超级权限           | 全部权限 + 权限分配、账号管理、敏感操作                          |

---

## **三、功能模块详细需求**

### **3.1 登录与注册模块**

### **3.1.1 登录页面**

* 账号密码登录
* 品牌名称：「嗯呢呗电竞」
* 登录后根据角色跳转不同首页

### **3.1.2 注册方式（两种）**

### **表格**复制

| **方式** | **流程**                                           | **默认身份** |
| -------------- | -------------------------------------------------------- | ------------------ |
| KOOK注册       | 填写KOOK ID → 选择申请铭牌（GOD/小猪崽）→ 设置账号密码 | 老板(GOD)或陪玩    |
| 微信注册       | 扫码关注公众号自动注册                                   | 老板(GOD)          |

**注册规则：**

* 账号：仅支持数字/字母
* 默认密码：123456789（KOOK注册时）
* 陪玩注册后账号名为「用户00001」格式，需管理员手动添加昵称和铭牌

---

### **3.2 用户中心（个人首页）**

### **3.2.1 信息展示**

* **身份铭牌显示** ：GOD（老板）、小猪崽（陪玩）、客服宝宝、管理宝宝、高级管理大人
* **匿名播报切换** ：充值播报、送礼物冠名、陪玩收礼物冠名均可设置匿名
* **账户余额** ：M币余额（充值货币）
* **消费经验/积分** ：显示当前积分，比例1:1
* **VIP等级** ：自动显示当前VIP等级及下单折扣
* **接单余额** ：M豆余额（陪玩收益），可跳转提现或转换余额

### **3.2.2 订单记录**

* **订单详细** ：日期、类别、详情、老板、陪玩、订单项目、订单时长、金额、更新余额
* **接单详细** ：日期、类别、详情、老板、陪玩、订单项目、订单时长、订单总额、到手总额
* **时间筛选功能**

---

### **3.3 后台首页（数据看板）**

### **3.3.1 数据统计卡片（管理可见）**

* 7日交易额 / 7日订单数量 / 客户总量 / 订单总量
* **时间筛选** ：日、周、月切换

### **3.3.2 升级记录（独立页面）**

* 字段：用户、等级、升级时间、权益发放状态
* 操作：客服确认发放权益
* 权限：仅客服和管理后台可见

### **3.3.3 待办事项**

* 待解冻冠名数量
* 待审提现数量

### **3.3.4 老板排行榜**

* 字段：头像、昵称、经验值、累计充值、累计赠金、累计消费
* **权限** ：仅管理后台可见，经验值需明显显示

### **3.3.5 客服数据（客服&管理可见）**

* 字段：昵称、派发订单数、订单分红、累计订单时长、派发礼物数、累计礼物分红
* **时间筛选** ：日、周、月、总数
* **计算规则** ：
* 派单订单数 = 此客服总共发了多少订单
* 累计订单时长 = 总共发的订单内的时长/局数总和
* 派发礼物 = 此客服累积总发了多少个礼物订单
* 累计礼物分红 = 总礼物订单金额的1%分红

---

### **3.4 工作打卡系统（独立页面）**

### **3.4.1 打卡功能**

* **开始上班** ：点击后开始计时
* **下班记录** ：点击后记录结束时间
* **状态保持** ：关闭网页仍然生效，4小时后自动结算下班
* **导出功能** ：可导出打卡记录，筛选具体日期

### **3.4.2 打卡记录表**

* 字段：开始时间（年月日时分秒）、结束时间、时长（自动计算）、客服（仅本人可见）

### **3.4.3 考勤规则**

* 以打卡上下班中间时长为主
* 缺失10分钟按正常客服规则算迟到/旷班
* 网页关闭后上班打卡仍有效，直至手动下班

---

### **3.5 用户管理（客户管理）**

### **3.5.1 用户列表**

* **搜索筛选** ：KOOK ID（精准/模糊）、客户昵称（模糊）、陪玩昵称（模糊）
* **快捷操作** ：KOOK注册、微信注册、导出EXCEL、导出全部数据
* **列表字段** ：头像、KOOK ID、客户昵称、陪玩昵称、绑定状态（微信/KOOK）、余额、佣金、等级、折扣、经验值、操作

### **3.5.2 等级体系**

* 默认注册等级：GOD（老板身份）
* 升级路径：GOD → VIP1-VIPn → 总裁sama等

### **3.5.3 单项用户管理（详情页）**

**基础资料标签：**

* KOOK ID + 可编辑陪玩昵称
* 余额、佣金、经验值卡片展示
* 第三方关联：KOOK绑定状态、微信绑定状态
* 更多信息：推荐人、匿名升级状态、状态、专属播报频道、标签、权益
* **权限操作（红框=管理可见，黑框=客服&管理可见）** ：
* 重置密码（管理）
* 设置推荐人（管理）
* 修改经验值（管理）
* 货币转换（管理）
* 切换升级匿名播报（客服&管理）
* 修改播报专属频道（客服&管理）
* 绑定微信（客服&管理）

**M币标签（余额日志）：**

* 筛选：充值、消费、赠送
* 日期筛选
* 手动变账功能（弹出窗：操作类型[充值/扣款/赠金]、变动金额、变动理由）
* **权限** ：扣款仅管理可见，充值和赠金客服管理可见
* **规则** ：充值余额消费可增加经验值，赠金余额消费不可增加

**M豆标签（佣金日志）：**

* 筛选：增加、扣除
* 字段：变动金额、变动理由、剩余金额、变动时间

**订单数据标签：**

* 筛选：详细日期
* 字段：订单号、陪玩昵称、游戏项目、实际价格、订单状态、派单客服、派单日期、详情按钮

**亲密度数据标签：**

* 亲密度列表：亲密度对象、亲密度值、操作（修改/删除）
* **功能** ：管理可修改/删除，自动按亲密度排序

**订单合并数据标签：**

* 订单合计数据：陪玩昵称、合计金额
* 礼物合计数据：陪玩昵称、累计送礼
* **规则** ：如若退款自动扣除

### **3.6 订单管理**

### **3.6.1 订单列表**

* **搜索筛选** ：订单号（精准）、老板昵称（精准）、陪玩昵称（精准）、派单客服（精准）、派单日期范围、订单状态
* **统计卡片** ：总订单额、陪玩工资、平台营收
* **快捷操作** ：立即派单、导出筛选数据
* **列表字段** ：订单号、老板昵称、陪玩昵称、游戏项目、实际价格、陪玩收益、订单状态、冻结状态、派单客服、派单日期、操作

### **3.6.2 订单状态**

* 待解冻、待支付、已支付、已退款、待申报

### **3.6.3 冻结状态**

* 已冻结 / 未冻结
* 冻结后佣金可见但无法提现

### **3.6.4 订单详情弹窗**

* 完整订单信息展示
* 退款确认：勾选「我已确认为该用户退款」后再次确认

### **3.6.5 立即派单**

**正常陪玩订单：**

* 字段：游戏项目（级联选择）、基础单价、补充单价、附加项目、备注、老板（模糊搜索）、陪玩（模糊搜索，仅显示陪玩昵称不带KOOK ID）
* **规则** ：
* 选择游戏项目后自动出现定好的单价
* 余额不够单价金额则发不了单
* 陪玩手动报单 → 老板手动确认 → 24小时后自动确认转换为陪玩佣金

**护航/代练派单（赠送订单模式）：**

* 基础单价默认为1
* 时长/数量字段
* **规则** ：发单后直接扣除老板后台余额到陪玩佣金内，需自动冻结，手动解冻

## **五、核心业务规则**

### **5.1 货币体系**

### **表格**复制

| **货币** | **用途**     | **来源**     |
| -------------- | ------------------ | ------------------ |
| M币            | 消费、下单、送礼物 | 充值、赠金         |
| M豆            | 陪玩收益、提现     | 接单完成、礼物收入 |
| 积分/经验值    | VIP等级提升        | 消费（充值余额）   |

### **5.2 订单流程**

### **plain**复制

`派单 → 陪玩接单 → 游戏完成 → 陪玩申报时长 → 老板确认/24h自动确认 → 佣金到账(M豆)`

### **5.3 冻结规则**

* 护航/代练/赠送订单：自动冻结，需手动解冻
* 异常订单：客服/管理可手动冻结
* 冻结状态：可见余额但无法提现

### **5.4 分红规则**

* 陪玩分红：按项目配置比例（默认80%）
* 客服礼物分红：礼物订单金额的1%

### **5.5 经验值规则**

* 充值余额消费：可增加经验值（比例1:1）
* 赠金余额消费：不可增加经验值

---

## **六、界面与交互要求**

1. **响应式设计** ：适配客服日常使用的各种屏幕尺寸
2. **操作确认** ：所有敏感操作（删除、退款、发放提现、修改权限）需二次确认
3. **数据导出** ：关键列表支持EXCEL导出
4. **实时刷新** ：关键数据（订单状态、余额）支持自动刷新
5. **权限提示** ：无权限功能隐藏或禁用，避免误操作
